# GeoJSON Loader 拡張機能 仕様書

| 項目 | 内容 |
| --- | --- |
| 拡張機能 ID | `omni.geojson.loader` |
| バージョン | 0.1.0 (MVP) |
| 対象 SDK | NVIDIA Omniverse Kit SDK 106.x |
| 対象アプリ | `kit-app-template` から生成した USD Composer ベースアプリ |
| 作成日 | 2026-05-26 |

---

## 1. 目的と背景

USD Composer 上で **GIS データ（GeoJSON）を USD ステージに取り込む**ための最小実装 (MVP) を提供する。
最初のリリースでは、ユーザーが手元の `.geojson` / `.json` ファイルをアップロード（選択）し、その中身を UI で確認したうえで、3D シーン上に USD Prim として可視化できる状態を目指す。

## 2. 用語

| 用語 | 説明 |
| --- | --- |
| **GeoJSON** | RFC 7946 で定義される地理空間データ用の JSON 形式。 |
| **Feature** | 1 つの地物。`geometry` と `properties` を持つ。 |
| **FeatureCollection** | 複数の Feature を束ねたコレクション。本拡張のメイン入力。 |
| **Prim** | USD のシーングラフ上の最小単位。Xform / Mesh / Points / BasisCurves など。 |
| **Stage** | 現在開いている USD シーン全体。`omni.usd.get_context().get_stage()` で取得。 |

## 3. システム構成

```
┌──────────────────────────────────────────────┐
│  USD Composer (kit-app-template)             │
│  ├─ omni.ui (UI 描画)                        │
│  ├─ omni.usd (Stage アクセス)                │
│  └─ omni.geojson.loader (本拡張)             │
│      ├─ extension.py     UI と lifecycle      │
│      ├─ geojson_parser.py 解析と検証          │
│      └─ usd_builder.py    USD Prim 生成      │
└──────────────────────────────────────────────┘
```

依存している Kit 標準拡張は以下のみで、追加の Python ホイールは要求しない。

- `omni.kit.uiapp`
- `omni.ui`
- `omni.usd`
- `omni.kit.menu.utils`
- `omni.kit.window.filepicker`（オプション）

## 4. 機能要件 (Functional Requirements)

### 4.1 必須機能 (MVP スコープ)

| ID | 機能 | 詳細 |
| --- | --- | --- |
| F-01 | ファイル選択 | `Window > GeoJSON Loader` を開くと、ファイルパス入力欄と `Browse...` ボタンを表示する。`omni.kit.window.filepicker` が利用可能であればネイティブのファイルピッカーを開く。 |
| F-02 | パース実行 | `Load & Preview` ボタンで指定ファイルを解析し、結果の要約 (件数・型別件数・bbox) を UI のプレビュー領域に表示する。 |
| F-03 | エラー表示 | 不正な JSON、必須プロパティ欠落、閉じていないポリゴンなどは `Status` 行に人間可読のエラーメッセージを表示する。例外発生時はトレースバックをプレビューに出力する。 |
| F-04 | 3D 可視化 | `Build on Stage` ボタンで、解析結果を現在の Stage の `/World/GeoJSON/<layer>` 配下に USD Prim として書き込む。 |
| F-05 | オプション設定 | UI からスケール、ポリゴン押し出し高さ、ルート Prim パス、経緯度→メートル変換の有無を変更できる。 |
| F-06 | プロパティ保存 | Feature の `properties` は対応する USD Prim に `userProperties:<key>` カスタム属性として保存する。 |
| F-07 | クリア | `Clear Preview` で現在のプレビュー状態をリセットする（Stage 上の Prim は削除しない）。 |

### 4.2 非機能要件 (Non-Functional Requirements)

| ID | 区分 | 内容 |
| --- | --- | --- |
| N-01 | 性能 | 10,000 Feature 程度（合計 100 万頂点未満）までを 5 秒以内に取り込めること。 |
| N-02 | 安定性 | 不正データを与えても拡張がクラッシュしないこと（捕捉不能例外は出さない）。 |
| N-03 | 移植性 | 追加の Python パッケージインストールなしに、`kit-app-template` 既定の `python.exe` で動作すること。 |
| N-04 | コード品質 | `python -m py_compile` で全 `.py` ファイルが構文エラー無くコンパイルできること。 |
| N-05 | 国際化 | UI ラベルは英語、ドキュメントは日本語とし、UTF-8 文字列の properties をそのまま保持できること。 |

### 4.3 スコープ外 (Out of Scope) — 将来検討事項

- 厳密な座標系変換（EPSG 指定、WebMercator、Lambert など） — 現状は等距円筒で代用。
- ポリゴンの内包穴 (interior rings) の正確な三角形分割 — 現状は外周のファン三角形分割。
- 巨大ファイルのストリーミング読み込み。
- Stage への上書き保護（同名 Prim がある場合に確認ダイアログを出す等）。
- マテリアル / シェーダ自動アサイン。

## 5. 入出力仕様

### 5.1 入力

| 項目 | 内容 |
| --- | --- |
| 形式 | RFC 7946 準拠の GeoJSON。拡張子は `.geojson` または `.json`。 |
| 文字コード | UTF-8。BOM 付きも許容 (`Path.read_text(encoding="utf-8")`)。 |
| 対応する `type` | `FeatureCollection`, `Feature`, `Point`, `MultiPoint`, `LineString`, `MultiLineString`, `Polygon`, `MultiPolygon`, `GeometryCollection`。 |
| 座標順 | `[longitude, latitude, (altitude)]` の順。 |

### 5.2 出力

入力 1 ファイルにつき、以下の構造の USD Prim を生成する。

```
/World/GeoJSON                  (Xform: ルート)
└─ /World/GeoJSON/<layer>       (Xform: ファイル単位)
   ├─ /World/GeoJSON/<layer>/f_<id>
   │  └─ geom                   (Points / BasisCurves / Mesh)
   ├─ /World/GeoJSON/<layer>/f_00001
   │  └─ geom
   ...
```

### 5.3 ジオメトリ対応表

| GeoJSON | USD Prim 型 | 備考 |
| --- | --- | --- |
| Point | `UsdGeomPoints` | `widths` 属性に `point_radius` を設定。 |
| MultiPoint | `UsdGeomPoints` | 同上、複数頂点。 |
| LineString | `UsdGeomBasisCurves` (linear) | `widths` 属性に `line_width` を設定。 |
| MultiLineString | `UsdGeomBasisCurves` (linear) | `curveVertexCounts` で複数ラインを束ねる。 |
| Polygon | `UsdGeomMesh` | 外周をファン三角形分割。内周（穴）は MVP では無視。 |
| MultiPolygon | `UsdGeomMesh` | 各 Polygon を順に頂点配列に追加し、`faceVertexCounts` で区切る。 |
| GeometryCollection | 子 `Xform` 配下に上記マッピングを再帰的に適用 | — |

### 5.4 座標変換

| モード | 説明 |
| --- | --- |
| `project_to_meters = true` (既定) | 入力 bbox の中心を原点とした等距円筒投影で経緯度をメートルに変換。X = 東, Z = -北, Y = 高度。 |
| `project_to_meters = false` | 経緯度をそのまま X/Z にマップ（テスト・デバッグ向け）。 |
| `scale` | 投影後の全座標に乗算する一様スケール。 |
| `polygon_height` | Polygon / MultiPolygon の Y 値に加算するオフセット（押し出し簡易表現）。 |

USD のデフォルト上方向は **+Y** なので、平面方向 (経度・緯度) を **X/Z** 平面、標高方向を **Y** にマップする。

## 6. UI 仕様

| 要素 | 種別 | 役割 |
| --- | --- | --- |
| ファイルパス | `StringField` + `Browse...` ボタン | 読込対象を指定。`omni.kit.window.filepicker` でダイアログ表示。 |
| Root prim | `StringField` | 出力ルートの USD パス。既定 `/World/GeoJSON`。 |
| Scale | `FloatDrag` | 全座標に乗算する倍率。既定 1.0。 |
| Polygon height (m) | `FloatDrag` | Polygon の Y 方向押し出し量。既定 0.0。 |
| Project lon/lat to meters | `CheckBox` | 経緯度→メートル変換の ON/OFF。既定 ON。 |
| Load & Preview | `Button` | パースして要約を表示。 |
| Build on Stage | `Button` | USD Prim として現在の Stage に書き込み。 |
| Clear Preview | `Button` | プレビューの内部状態をリセット。 |
| Status | `StringField (read_only)` | 1 行ステータス（Idle / Parsed N features / Error …）。 |
| Preview / Summary | `StringField (multiline, read_only)` | 解析結果の JSON サマリと、エラー時のトレースバックを表示。 |

UI は `Window > GeoJSON Loader` メニューから開く。
初回表示時に `Property` パネルにデフォルトでドックする。

## 7. データモデル (`geojson_parser.py`)

```python
@dataclass
class Geometry:
    type: str
    coordinates: Any            # 型ごとに正規化された形状で格納
    geometries: List[Geometry]  # GeometryCollection のみ使用

@dataclass
class Feature:
    geometry: Optional[Geometry]
    properties: Dict[str, Any]
    id: Optional[Union[str, int]]

@dataclass
class FeatureCollection:
    features: List[Feature]
    bbox: Optional[Tuple[float, float, float, float]]
    source_path: Optional[str]
```

`coordinates` の正規化フォーマットは以下のとおり。

| Geometry.type | `coordinates` の型 |
| --- | --- |
| Point | `(x, y, z)` |
| MultiPoint, LineString | `List[(x, y, z)]` |
| MultiLineString | `List[List[(x, y, z)]]` |
| Polygon | `List[List[(x, y, z)]]` （外周＋穴のリング配列） |
| MultiPolygon | `List[List[List[(x, y, z)]]]` |
| GeometryCollection | `coordinates = None` / `geometries` を使用 |

## 8. 設定キー

`extension.toml` で定義する既定値。アプリ側 `.kit` ファイルから上書き可能。

| キー | 既定値 | 用途 |
| --- | --- | --- |
| `exts."omni.geojson.loader".default_height` | `0.0` | Polygon 押し出しの初期値 |
| `exts."omni.geojson.loader".default_scale` | `1.0` | Scale の初期値 |
| `exts."omni.geojson.loader".default_root_path` | `/World/GeoJSON` | Root prim の初期値 |
| `exts."omni.geojson.loader".project_to_meters` | `true` | 投影 ON/OFF の初期値 |

## 9. エラーハンドリング

| 状況 | 動作 |
| --- | --- |
| ファイルが存在しない / 読めない | `GeoJsonError` → Status に "Parse error: …" 表示 |
| JSON 形式が不正 | `GeoJsonError` → 同上 |
| 未対応ジオメトリ型 | `GeoJsonError` → 同上 |
| Polygon の閉ループ違反 | `GeoJsonError` → 同上 |
| Stage が存在しない | `RuntimeError` → "Build error: No active USD stage. …" |
| 個別 Feature の USD 化に失敗 | その Feature のみスキップし、`geojson:error` 属性に理由を記録（カウント `skipped` に加算） |

## 11. リスクと前提

- 等距円筒投影は赤道付近では十分な近似だが、高緯度では歪みが大きい。実運用では `pyproj` 等の導入を検討。
- ポリゴンの単純ファン三角形分割は凸ポリゴン以外で破綻し得る。Earcut ライブラリの組み込みを将来検討。
- 非常に大きな GeoJSON（数十万 Feature）の読み込みパフォーマンスは MVP では考慮しない。
