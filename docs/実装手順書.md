# GeoJSON Loader 拡張機能 実装手順書

本書は `omni.geojson.loader` 拡張機能を `kit-app-template` ベースの **USD Composer** アプリへ組み込み、起動・動作確認を行うまでの手順をまとめたものです。

> 仕様の詳細は `仕様書.md` を参照してください。
> 本書はあくまで「組み込む側の作業」にフォーカスします。

---

## 0. 前提条件

| 項目 | 推奨 |
| --- | --- |
| OS | Windows 10/11 または Ubuntu 22.04 |
| GPU | RTX A4000 以上（USD Composer の動作要件に準ずる） |
| Kit SDK | 106.x (kit-app-template 既定) |
| Python | Kit 同梱 (3.10 系)。追加の `pip install` は不要 |
| Git | 必須（kit-app-template を取得するため） |

NVIDIA Launcher 経由で Omniverse をインストール済みで、`kit-app-template` リポジトリを取得できる状態であることを前提にします。

---

## 1. kit-app-template の取得とアプリ生成

1. 任意のディレクトリで `kit-app-template` を取得します。

   ```bash
   git clone https://github.com/NVIDIA-Omniverse/kit-app-template.git
   cd kit-app-template
   ```

2. テンプレートから USD Composer ベースのアプリを生成します。

   ```bash
   # Windows
   .\repo.bat template new

   # Linux
   ./repo.sh template new
   ```

   プロンプトで `Application` → `USD Composer` を選択し、アプリ名 (例: `my_company.usd_composer`) を入力します。

3. 生成後のディレクトリ構成は概ね以下のようになります（抜粋）。

   ```
   kit-app-template/
   ├─ source/
   │  ├─ apps/
   │  │  └─ my_company.usd_composer.kit
   │  └─ extensions/        ← ここに本拡張を配置
   ├─ tools/
   └─ repo.toml
   ```

---

## 2. 拡張機能のコピー

本リポジトリ（`omni.geojson.loader/`）一式を、生成された `source/extensions/` 配下にコピーします。

```bash
cp -r omni.geojson.loader  <kit-app-template>/source/extensions/
```

最終的な配置：

```
source/extensions/omni.geojson.loader/
├─ config/
│  └─ extension.toml
├─ data/
│  └─ sample.geojson
├─ docs/
│  ├─ 仕様書.md
│  └─ 実装手順書.md
├─ omni/
│  └─ geojson/
│     └─ loader/
│        ├─ __init__.py
│        ├─ extension.py
│        ├─ geojson_parser.py
│        └─ usd_builder.py
└─ tests/
   └─ test_geojson_parser.py
```

---

## 3. アプリ `.kit` ファイルへの登録

`source/apps/my_company.usd_composer.kit` を開き、`[dependencies]` セクションに本拡張を追記します。

```toml
[dependencies]
# ... 既存の依存 ...
"omni.geojson.loader" = {}
```

必要に応じて起動時の設定値も上書きできます。

```toml
[settings."exts.omni.geojson.loader"]
default_root_path = "/World/GIS"
project_to_meters = true
default_scale     = 1.0
```

---

## 4. ビルドと起動

1. ビルド

   ```bash
   # Windows
   .\repo.bat build

   # Linux
   ./repo.sh build
   ```

   ビルドは `_build/` 配下に生成されます。

2. 起動

   ```bash
   # Windows
   .\repo.bat launch -n my_company.usd_composer

   # Linux
   ./repo.sh launch -n my_company.usd_composer
   ```

   起動後、メニューバーの `Window > GeoJSON Loader` を選択し、ウィンドウが表示されることを確認します。

---

## 5. 動作確認（受け入れ確認）

以下の手順で MVP として最低限の動作が満たされていることを確認します。

1. **新規ステージを作成**
   `File > New From Stage Template > Empty` などで空のステージを開く。

2. **GeoJSON Loader を開く**
   `Window > GeoJSON Loader`。

3. **サンプルを読み込む**
   `Browse...` から `source/extensions/omni.geojson.loader/data/sample.geojson` を選択。

4. **`Load & Preview` を押す**
   Status に "Parsed 3 feature(s) from sample.geojson" と表示され、Preview に JSON サマリが出力されること。

   ```json
   {
     "feature_count": 3,
     "by_type": {
       "Point": 1,
       "LineString": 1,
       "Polygon": 1
     },
     "bbox": [139.745, 35.6586, 139.7588, 35.685],
     "source": ".../sample.geojson"
   }
   ```

5. **`Build on Stage` を押す**
   Stage 上の `/World/GeoJSON/sample` 配下に Point / BasisCurves / Mesh の Prim が生成されること。
   ビューポートで対象 Prim にフォーカス (`F` キー) すると、Tokyo Tower 付近を中心に 3 つの地物が確認できます。

6. **`Clear Preview` を押す**
   プレビューが空になり、Status が "Cleared." になること（Stage の Prim はそのまま残ります）。

7. **不正データのテスト**
   メモ帳で `{ "type": "FeatureCollection", "features": [` のみのファイルを作り、読み込むと Status に "Parse error: …" が表示されること（アプリは落ちない）。

## 6. デバッグ Tips

| 状況 | 対応 |
| --- | --- |
| メニューに項目が出ない | `Window > Extensions` で `omni.geojson.loader` が **Enabled** になっているか確認。 |
| `ImportError: omni.kit.window.filepicker` | Kit のバージョンが古い可能性。拡張は手動パス入力にフォールバックします。 |
| Prim が原点付近に集まる | `Project lon/lat to meters` を ON にして bbox 中心で投影されることを確認。 |
| Prim が極端に大きい / 小さい | `Scale` 値を変更（例: 0.01 / 100）。 |
| Polygon が真っ平ら | `Polygon height (m)` を 1〜数十に設定。 |

ログは USD Composer 起動時のターミナル、または `Window > Script Editor > Logging Window` で確認できます。

---

## 7. ユニットテスト

ステージなしのパーサ・投影ロジックは標準 Python だけでテストできます。

```bash
cd source/extensions/omni.geojson.loader
python -m unittest discover -s tests
```

期待結果: `OK (5 tests)` 程度。
失敗時はサンプルファイルが破損していないか、`PYTHONPATH` が `omni.geojson.loader/` を指しているかを確認。

Kit 環境上でのテストは `omni.kit.test` を用いて将来追加します（`extension.toml` の `[[test]]` セクションでフックは準備済み）。

---

## 8. 配布

1. `omni.geojson.loader/` 全体を zip にし、社内アーカイブに保管。
2. もしくは `kit-app-template` のリポジトリにコミットし、`repo.sh package` で配布用パッケージを作成。
3. 本拡張は外部ホイールに依存しないので、追加のオフラインインストール手順は不要です。
